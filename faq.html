Discuss the scope of var, let, and const?

1. var: "var" keyword is a global scope. That means you can access a "var" variable before it is declared in code. You can declare the same variable name multiple times within the same scope without error.

2. let: "let" keyword is a Block-scope. That means you can not declare a "let" variable with the same name in the same block-scope. "var" and "let" variables are hoisted,but they are not initialized until the actual declaration is encountered in the code.

3. const: "const" keyword is a Block-scope. That means you can not declare a "const" variable with the same name in the same block-scope. "const" requires an initial value when declared, and you can not change the value "const" variable after it is assigned.But if the value is an object,you can modify the properties of the object.

Tell us the use cases of null and undefined?

The use case of null and undefined is given below:-

1. undefined: Variables that are declared but not initialized explicity are assigned the value 'undefined' by default. If a function doesn't return a value explicity,it returns 'undefined' by default. If you try to access an object property that doesn't exist, it returns 'undefined'. Access an array element that doesn't exist will also returns 'undefined'.

2. null: 'null' is a value that is explicitly used to indicate the absence of a value. It can be used to reset the value of a variable or property to nothing, especially when you want to clear a reference.You can use 'null' in conditional statements to check if a value intentionally lacks a value.

What do you mean by REST API?

1. REST API stands for Representational State Transfer Application Programming Interface. It is a set of architectural constraints and principles for designing networked applications. REST is not a protocol but rather a style of designing networked applications that use the HTTP protocol.

2. REST APIs have a uniform and consistent interface, which simplifies interactions between clients and servers. This interface typically includes the use of standard HTTP methods (GET, POST, PUT, DELETE) and standard HTTP status codes (e.g., 200 OK, 404 Not Found) for indicating the outcome of requests.

3.RESTful APIs are widely used in web development and are the foundation of many web services, including those used for web and mobile applications. They provide a standard way for clients to interact with resources over the internet in a simple and consistent manner.
